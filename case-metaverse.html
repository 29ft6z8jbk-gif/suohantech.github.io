咪咕视频直播 PQC 质量监控中心集成与交付

发布时间：2024-05-15  |  业务领域：广播电视与融媒体  |  文章来源：《影视制作》

文章来源：《影视制作》 文/咪咕视讯科技有限公司 姚少康 张益维

引言：随着咪咕视讯业务版图扩张，大量的体育赛事信号、Liveshow 现场直播、二台制作等直播形态的业务全面铺开，其中境外体育赛事等外来信号以及节目制作过程具有极高的即时性，存在诸多不确定因素和安全隐患，为避免直播过程中出现意外和敏感信息，需要在较短的延时时长范围内降低直播播出风险，处理好直播信号的质量监控问题，并以最快最直接的方式进行应急处理，完成安全垫播片或广告片的插播功能。因此咪咕直播 PQC 质量监控中心需要在直播过程中通过对画面和声音的判断，对敏感内容作出快速响应，切换安全垫片，保障直播内容的安全播出。

我公司联合咪咕视讯打造了具备 5.1.4 监听能力的 4K 超高清 PQC 质量监控中心，为众多体育赛事的安全播出保驾护航。

1、系统组成和设计思路

为了满足咪咕直播 PQC 质量监控中心业务需求和技术目标，整个系统底层采用 4K-12G 基带架构，向下兼容 HD 模式。系统内信号格式为：分辨率 3840×2160，帧率 50P，支持 HDR 高动态范围，HLG 和 BT.2020 宽色域。最大满足 5 个工位主备信号同时并发监控的要求。系统从技术标准上仍然适配咪咕现有演播室集群 4K SMPTE 2110 的 IP 技术架构，符合 IP+ 基带的设计理念。

图1：系统整体流程与底层架构示意图

2、系统组成

咪咕直播 PQC 质量监控中心主要由 6 个子系统组成，包括信号 I/O 处理系统、延时系统、信号调度及应急切换系统、监听监看系统、垫片回放系统、通话系统。

图2：系统组成示意图

信号 I/O 处理系统包含编解码系统和 IP 视音频转换系统，主要负责系统内信号输入与输出，以及不同 IP 信号格式与基带信号之间的互相转换；延时系统对系统内信号进行延时处理，负责信号的延时时长范围设定；监听监看系统，负责系统内所有信号显示呈现，支持多画面分割技术；支持系统内时钟信号、画分源名和音频电平的显示；信号调度及应急切换系统负责内部所有信号的调度分配及播出信号的应急处理，在突发情况下，可提供最快最直接的播出应急保障；垫片回放系统支持实时录制和慢动作回放以及循环垫片播放，为播控系统提供稳定、可靠的安全替换内容；通话系统满足咪咕播控室与演播室、调度中心及总控之间的双向通话功能。

图3：系统整体信号流向图

3、系统工作原理与设计特点

直播 PQC 质量监控中心的工作原理，是以基带矩阵作为整个系统架构的核心枢纽，IP 信号源经过信号 I/O 处理系统转换为基带信号进入矩阵，由矩阵进行信号分配和路由的指派，其中将需要进行延时的信号调度至延时系统，并设定有效的延时时长，延时后的信号再环出作为矩阵的输入源指派至下一级信号 I/O 处理系统，完成信号的播出。

在整个过程中，监控员只需要监看和监听延时前的画面和声音，对直播内容进行判断，如需进行信号的应急播控，监控员可在有效的延时时长范围内，通过矩阵应急面板执行切换操作，将安全垫片替换延时后的直播信号，以此实现实时的内容播控，待直播内容完全脱敏后再将延时后的直播信号恢复切回。

技术亮点与核心优势：

  • 信号格式高兼容性：采用基带 + IP 的组合形式，兼顾通用互联网流媒体 IP 协议与 ST 2110 无压缩 IP 信号，使 I/O 结构更为灵活多变。
  • 多路并发监控：同时满足 5 个独立的监控工位，每个工位具备独立的监听、监看环境，支持 10 路主备信号并发监控。
  • 高冗余与安全性：采用多级冗余架构，每级子系统的核心设备主备冗余，子系统间为双链路结构配置，并配备主备冗余电源自动倒换。

4、系统设计方案总结

直播 PQC 质量监控中心是咪咕视讯自主设计，与上海索涵智能科技有限公司共同搭建开发的一套基于直播态的内容质量监控系统。它贴合咪咕自有演播室实际业务情况，是实现咪咕视讯内容生产流程中直播信号延时播控、内容质量监控的核心系统之一。

该系统实现了咪咕视讯对于演播室生产流程中质量监控环节的全面覆盖，做到不留盲区、不留死角、对问题“零容忍”、“零出现”。自系统建设以来，运行稳定，顺利完成了上千场播控监控，大大提升了直播业务的安全性，确保了播出的顺利进行。
