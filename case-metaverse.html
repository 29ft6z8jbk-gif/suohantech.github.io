1、系统组成和设计思路：

为了满足咪咕直播延时审核系统业务需求和技术目标，整个系统底层采用 4K-12G 基带架构，向下兼容HD 模式。系统内信号格式为 ：分辨率 3840×2160，帧率 50P，支持 HDR 高动态范围，HLG 和 BT.2020宽色域。最大满足 5 个工位主备信号同时并发审核的要求。系统从技术标准上仍然适配咪咕现有演播室集群4K SMPTE 2110 的 IP 技术架构，符合 IP+ 基带的设计理念。

2、系统组成
咪咕直播延时审核系统主要由 6 个子系统组成，包括信号 I/O 处理系统、延时系统、信号调度及应急切换系统、监听监看系统、垫片回放系统、通话系统。

系统示意图

信号 I/O 处理系统包含编解码系统和 IP 视音频转换系统，主要负责系统内信号输入与输出，以及不同 IP信号格式与基带信号之间的互相转换 ；延时系统对系统内信号进行延时处理，负责信号的延时时长范围设定 ；监听监看系统，负责系统内所有信号显示呈现，支持多画面分割技术；支持系统内时钟信号、画分源名和音频电平的显示 ；信号调度及应急切换系统负责内部所有信号的调度分配及播出信号的应急处理，在突发情况下，可提供最快最直接的播出应急保障 ；垫片回放系统支持实时录制和慢动作回放以及循环垫片播放，为播控系统提供稳定、可靠的安全替换内容 ；通话系统满足咪咕播控室与演播室、调度中心及总控之间的双向通话功能。

3、系统工作原理
直播延时审核系统的工作原理，是以基带矩阵作为整个系统架构的核心枢纽，IP 信号源经过信号 I/O 处理系统转换为基带信号进入矩阵，由矩阵进行信号分配和路由的指派，其中将需要进行延时的信号调度至延时系统，并设定有效的延时时长，延时后的信号再环出作为矩阵的输入源指派至下一级信号 I/O 处理系统，完成信号的播出。在整个过程中，审核员只需要监看和监听延时前的画面和声音，对直播内容进行审核和判断，如需进行信号的应急播控，审核员可在有效的延时时长范围内，通过矩阵应急面板执行切换操作，将安全垫片替换延时后的直播信号，以此实现实时的内容播控，待直播内容完全脱敏后再将延时后的直播信号恢复切回。

设计特点
（1）信号格式高兼容性
考虑到前审和后审模式下，输入信号和播出目标的信号格式均是不同的，在设计整个系统架构时需要多方位考量，既能满足市场上通用的互联网流媒体 IP 协议，也能支持 ST 2110 无压缩的 IP 信号，并将 IP 信号与基带信号打通，实现信号接口的统一与覆盖。相较于传统单一基带信号接口的方式，咪咕直播延时审核系统所采用的基带 +IP 的组合形式，大大提升了整体系统兼容性，使整个审核系统的 I/O 结构更为灵活多变。
（2）多路并发审核直播延时审核系统需同时满足 5 个独立的审核工位，每个工位具备独立的监听、监看环境。系统同时支持 10 路主备信号并发审核能力、每个工位支持同时对主备信号做应急切换操作。具备独立审核模式，互不干扰，针对前审模式和后审模式，每个工位可单独进行配置，系统具备快速调用及部署能力。
（3）冗余设计
面对高频率的直播需求，咪咕直播延时审核系统作为播出环节的最后一道重要关卡，其安全性尤为重要，这就要求无论在系统架构层面还是设备选型层面都要慎重考量，做到系统设计安全可靠，逻辑链路清晰明了，设备选择成熟稳定，保证系统能长时间稳定运行，与此同时系统具备足够的冗余备份能力和简单有效的应急处理机制，保障播出安全。为保障审核系统的高可靠性、高安全性，系统采用多级冗余的总体架构，每级子系统的核心设备主备冗余，除此之外在子系统与子系统之间设计为双链路结构配置模式，避免单个设备、单个链路发生故障时，导致系统瘫痪，确保信号播出的安全稳定。电力保障方面，延时审核系统配套机房的核心设备均配备了主备冗余电源，可实现主备电路的自动倒换，确保了可靠、稳定的电力供应。

4、 系统设计方案
4.1 信号I/O处理系统
咪咕直播延时审核系统的信号主要由两部分组成，第一部分为咪咕演播室集群核心数据交换中心的 IP 信号，即 ST 2110 无压缩信号 ；第二部分为咪咕总控的IP 流信号，如 RTMP、SRT 等深压缩信号。由于我们的矩阵是基带接口，通过在矩阵前、后级均部署 IP 视
音频转换系统（IPG）和编码器集群，确保咪咕演播室集群（ST2110）和咪咕总控（RTMP/SRT）的信号可以转换为基带信号进出核心矩阵，实现系统内基带信号与 IP 信号的互联互通。
具体的 I/O 链路以后审模式为例，演播室制作后的PGM 信号（无压缩的 ST2110 信号），通过 SONY的 LSM 系统调度至 IP 视音频转换系统（IPG），转换为基带信号后进入核心矩阵，矩阵内部调度指派至编解码集群，最终编码为 RTMP/SRT 流传输至咪咕总控，至此信号完成端到端的路由。在前审模式下，信号路由和 I/O 转换方式与后审模式恰恰相反。
4.2 延时系统
延时系统采用波视 BD30AV-UHD4 系列 4K 超高清延时器，支持 12G-SDI 输入、输出，延时时长范围最低可以达到 1 帧，最高支持到 30S。延时器具备 Bypass 直通功能，可以通过一键触发实现信号的旁通。系统中配置了 5 台双通道延时器全部环接于核心矩阵，以“并联”的方式，将延时器输入口、输出口、LOOP 口与矩阵接口固定相连，通过矩阵调度可对系统中任意信号进行延时，使得整个系统对于延时信号的调用非常灵活，可控。
4.3 信号调度及应急切换
在直播延时审核系统架构中信号的调度与路由指 派 是 由 矩 阵 来 实 现 的， 我 们 这 次 采 用 的 是 草 谷（GRASS VALLY）公司的基带矩阵。该矩阵支持 4K 12G-SDI 信号接口，最大可支持 48×48 输入输出通道，同时支持 48×12 画分监看交叉打点功能，在不占用额外输入、输出端口的前提下，可以调用矩阵中所有输入、输出信号进行画分的配置。矩阵的接口数量能满足 5 个审核工位的全量信号。
在硬件层面上部署两台配置完全相同的主备矩阵，主体为两台 2U 机箱，主、备独立工作，分别具备独立播出能力。主备矩阵之间也会预留足够的输入和输出接口，将二者互联，确保主矩阵中的主路信号提供给备矩阵，备矩阵中的备路信号提供给主矩阵，以此满足画分分配和调度的特殊使用需求。软件配置上可以实现主、备矩阵完全镜像配置或者部分独立操作配置。矩阵应急面板，可快速实现目标输入口指派至目标输出口的调度操作，同时搭配该面板配套的 Salvos 软件的预设功能，通过对应急切换面板上任意按键进行自定义，一键触发完成多个矩阵交叉点指令的执行。依附于 Salvos（宏命令）的功能，在实际调测和技术论证过程中给予我们极大的帮助和便捷性。例如在实际操作中，前审和后审的不同模式下，矩阵的交叉打点和应急切换设置是完全不同的，如果每次都需要人为手动对每一个交叉通道进行配置，再执行输
入、输出的切换，将耗费大量时间，并且人为反复操作也存在极大的风险隐患，而通过 Salvos 预设好的宏指令，可以让播控过程中的应急切换变得更为便捷、高效、安全。
4.4 监听监看系统
咪咕直播延时审核系统在直播的过程中需要同时监看多路画面，我们使用的是矩阵自带的画面分割技术，通过软件进行画分的配置。系统支持主、备两路独立的画分监看。主路画分监看由主矩阵提供，备路画分监看由备矩阵提供，确保系统链路冗余。根据信号监看要求，画面布局包含主备延时前的信源、主备延时后的信源、垫片信号及主备播出信号等。主、备路画分采用镜像设置，亦可独立设置，用于不同信号的对比监看。画分支持源名显示和音频电平显示。
在不同审核模式下，画分显示的信号源是不同的，通过 Salvos 设置，提前将多级指令进行预设，同样可以快速实现不同业务场景的画分更换，大大提升了审核业务模式的切换效率。监听部分，系统配置了立体声监听单元，支持 12-SDI 接口，可直接从矩阵调取任意信号进行监听，且无需再做解嵌处理。审核员在审核过程中全程监听延时前信号的声音，监听单元支持扩声或头戴式耳机独立监听。
4.5 垫片管理系统
垫片管理系统由两部分组成，分别为一台循环垫片设备和五台实时制作慢动作服务器。二者均接入主备矩阵，供审核人员应急切换，其中一台循环垫片设备作为系统中共用信源，同时提供给 5 个审核工位复用。出于稳定性和可靠性考虑，循环垫片设备采用了FPGA 架构的设备，支持 4K 12G-SDI 接口，最高支持 4K 50/60P 的录制和回放，且支持高动态范围 HDR及 HLG 曲线。循环垫片设备支持 Web 控制界面，支持远端对素材进行统一的管理和播出控制。此次我们也使用了轻量化的 vMIX 服务器作为实时收录和慢动作回放处理，相较于传统的循环垫片播放的宣传片或广告片，vMIX 可以将比赛直播的精彩画面实时录制后，进行快速剪辑，并制作成集锦或慢动作进行回放处理；亦可将一些现场空镜、观众反应镜头进行录制与回放。这种高度契合比赛内容的垫片播放选择，不但可以作为系统中循环垫片的补充与备份，还可以很好规避应急切换时，因突然插入的广告片或通用垫片，所导致的画面和声音“跳跃”而产生的生硬感和不适感。
4.6 通话系统
直播延时审核系统中部署了一套 Riedel 公司的通话面板与咪咕现有演播室集群使用的型号保持统一，实现系统内总控、演播室调度中心、导控室等区域的全双工通联。当通话面板发生故障时，审核工位配有无线通话腰包，可继续保持与其他各工位之间正常通话。

5.1.4 4K审片室全貌

直播延时审核系统是咪咕视讯自主设计，与上海索涵共同搭建开发的一套基于直播态的内容安全审核系统。它贴合咪咕自有演播室实际业务情况，是实现咪咕视讯内容生产流程中直播信号延时播控、内容安全审核的核心系统之一。该系统实现了咪咕视讯对于演播室生产流程中内容安全审核环节的全面覆盖，做到不留盲区、不留死角、对意识形态问题“零容忍”“零出现”。自系统建设以来，运行稳定，顺利完成了 上千场播控审核，大大提升了直播业务的安全性，确保了播出的顺利进行。
